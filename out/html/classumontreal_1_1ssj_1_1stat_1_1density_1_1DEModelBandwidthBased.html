Detailed Description

Implements a parametric model that is particularly designed for the estimation of the density of a random variable \(X = g(\mathbf U)\), where \(\mathbf U = (U_1,U_2,\dots,U_d)\sim U(0,1)^d \) with a given function \(g:(0,1)^d\rightarrow\mathbb{R}\) over a finite interval \([a,b]\).

Such situations arise naturally in uncertainty quantification. The observations \(X_0,X_1,\dots,X_{n-1}\) which are used to construct the density will typically be generated by simulation of \(\mathbf U\), either by Monte Carlo (MC) or by randomized quasi-Monte Carlo (RQMC).

The model applies to density estimators that rely on the selection of a bandwidth (or binwidth) and is currently implemented for histogram estimators, see DEHistogram, and kernel density estimators (KDEs), see DEKernelDensity. An extension to other bin-/bandwidth based estimators is easy to obtain as is explained further below. The model is thoroughly investigated in (TODO: cite our paper) and can be explained as follows. It is known in general, that the mean integrated square error (MISE) can be rewritten as the sum of the integrated variance (IV) and the integrated square bias

\[ \textrm{MISE} = \int_a^b\mathbb{E} [\hat{f}_{n,h}(x) - f(x)]^2\mathrm{d}x = \int_a^b\textrm{Var}[\hat{f}_{n,h}(x)]\mathrm{d}x + \int_a^b \left( \mathbb{E}[\hat{f}_{n,h}(x)] - f(x) \right)^2\mathrm{d}x = \textrm{IV} + \textrm{ISB}, \]

where \(f\) denotes the true density and \(\hat{f}_{n,h}\) the density estimator. For MC it is known that, asymptotically, \(\textrm{IV}\approx C n^{-1} h^{-1}\) and \(\textrm{ISB}\approx B h^{\alpha}\), \(C,B,\alpha>0\). Observe that the power of \(h\) is positive in the ISB, while it appears to be negative in the IV.

The goal is to select an optimal band- or binwidth \(h_*\) which balances these two terms. This is also known as the infamous variance-bias tradeoff. One can do so, as soon as all the parameters \(C,B,\alpha\) are explicitly known or can be sufficiently well estimated, at least. For histograms and KDEs this is indeed the case. Let \(R(g)\) denote the roughness functional over the interval \([a,b]\) and let \(\mu_k(g)\) be the \(k\)-th moment of a function \(g\), i.e.

\[ R(g) = \int_a^bg^2(x)\mathrm{d}x,\qquad\text{and}\quad \mu_k(g)=\int_{-\infty}^{\infty}x^kg(x)\mathrm{d}x. \]

Assuming that all the entities exist and are finite, the table below lists the parameter values for histograms and KDEs for observations generated with MC, see [202].

Asymptotically optimal parameter values with MC
Parameter Histogram KDE
\(C\) 1 \(\mu_0(K^2)\)
\(\alpha\) 2 4
\(B\) \(R(f')/12\) \( \mu_2^2(K) R(f'')/4\)

The only highly unpleasant quantities are the roughness functional of the first or second derivative of the unknown density. This, however, can be overcome by using derivative estimates as implemented in DensityDerivativeEstimator and plugging the results back into the expression for the ISB. Here, the kernel selected for the derivative estimation is a standard normal. To estimate the \(h^{r_0}_{\text{AMISE}} \), where \(r_0\in\{1,2\}\), as can be seen in the table above, we use one step of the recursion DensityDerivativeEstimator::hAmiseR(int, int, double, double[], double, DensityDerivativeEstimator, double[], double[], double, double). For the initial value we estimate the empirical standard deviation \(\hat{\sigma}\) from the observations \(X_0,\dots,X_{n-1}\) and assume \(f\) to be a normal distribution with standard deviation \(\hat{\sigma}\).

If the user wishes to use different kernels or other initial values than those stated above, it is only necessary to override the corresponding methods for estimating \(B\). The rest will remain intact.

Let us now turn to the IV. What changes with RQMC is that it can cause significant variance reduction, which, however, can come at the price of a worse dependency on \(h\). This was proven theoretically and also observed empirically in (TODO: cite our paper). To reflect this behavior, we choose the model

\[ \textrm{IV}\approx C n^{-\beta}h^{-\delta},\qquad \beta,\delta>1, \]

locally, i.e. within a small region w.r.t. \((n,h)\). This proved to be a reasonable assumption.

Observe that taking the logarithm of both sides of the above model assumption yields a linear model with variables \(\log n\) and \(\log h\), and unknown parameters \(\log C,\beta,\delta\). Based on this linear model, one can estimate the empirical IV for several reasonable values of \((n,h)\) with the methods provided by DensityEstimator and subsequently obtain estimates for the unknown parameters. Together with the estimated parameters from the ISB, one can thus compute an estimate for \(h_*\)

\[ h_* = \kappa n^{-\gamma},\qquad\text{with }\kappa = \left(\frac{C \delta}{B \alpha} \right)^{1/(\alpha + \delta)},~\gamma = \frac{\beta}{\alpha+\delta}, \]

as well as the modeled MISE

\[ \textrm{MISE} \approx Kn^{-\nu} ,\qquad\text{with } K = C \kappa^{-\delta} + B \kappa^{\alpha},~\nu=\frac{\alpha\beta}{\alpha+\delta}. \]

The main method of this class is #testMISERate which, in turn, is made up of three main components. These are #parametersIVestimate, parametersISBEstimate, and #estimateMISEOptH. The first component uses a test grid w.r.t. \((n,h)\) at which it estimates the empirical IV and, subsequently, estimates the IV-parameters by linear regression. The second component carries out the task of computing the parameter \( B\) as described above. The method #estimateMISEOptH computes the optimal bin-/bandwidth \(h_*\) and runs the experiment for the optimal bin-/bandwidth with newly generated observations. It needs to be added that two flags can be set, which are set to false by default. If true, displayExec prints output during the runtime of the program and producePlots generates plots at the end of the program. The output of all the variants of the main method is a formatted string containing a detailled report of the entire experiment.

Finally, if the user wants to use this class for other estimators than DEHistogram or DEKernelDensity this can easily be achieved by implementing the methods setH(DensityEstimator, double), setAlpha(DensityEstimator), and parametersISBEstimate specifically for this estimator.

Estimates the parameter \(B\) according to the formula from the table given above.

This is done by generating \(n\) observations of model with the RQMC points rqmc. It computes the empirical mean \(\hat{\mu}\) and standard deviation \(\hat{\sigma}\) of these observations and uses them to obtain an initial value for the roughness functional of the sought density of order order + 2 over \([a,b]\). This initial value is computed as the roughness functional of the density of a normal distribution with parameters \((at{\mu},\hat{\sigma})\). It then iterates once over the recursive formula for the optimal bin-/bandwidth given in umontreal.ssj.stat.density.DensityDerivativeEstimator with umontreal.ssj.stat.density.DEDerivativeGaussian::hAmiseR. Using this optimal bin-/bandwidth, it estimates the derivative of order order of the sought density by a umontreal.ssj.stat.density.DEDerivativeGaussian.

This procedure is repeated \(m\) times, each time with rqmc randomized afresh and the mean of the result is set as \(B\)s.

Note that, for estimating B order has actually to be 1 or 2. This is implicitly assumed, as setInitRoughnessFunctional would throw an exception here for any other order.

Parameters
modelthe underlying model.
rqmcthe RQMC point set used.
mthe number of independent replications.
evalPointsthe evaluation points to estimate the roughness functionals.
orderthe sought order of the derivative of the density. Has to be 1 or 2 for a histogram or a kernel density estimator, respectively.
factorthe factor with which the respective roughness functional is multiplied to yield \(B\). For a histogram this is 1/4 and for a KDE 1/12.

◆ parametersIVEstimate() [2/2]

String parametersIVEstimate ( MonteCarloModelDouble  model,
RQMCPointSet []  rqmcPts,
int  m,
DensityEstimator  de,
double []  hArray,
double []  evalPoints 
)

This method is the core function for estimating the IV parameters \(C\), \(\beta\), and \(delta\) by estimating the empirical IV over a testing region for \((n,h)\) and, subsequently, performing a linear regression.

More precisely, rqmcPts is intended to contain the same type of point set in different sizes \(n\). These sizes are the \(n\)-values for the testing region. The \(h\)-values are passed in hArray. For every point in rqmcPts it generates observations of model. This is repeated \(m\) times independently. On the basis of these observations it estimates the empirical IV for every \(h\) in hArray. The evaluation points used to estimate the empirical IV are passed in evalPoints The estimated IVs are stored in logIV3D. Finally, this data is used to estimate the parameters by a linear regression.

The string which is returned is composed of an introductory heading, a table showing the results of the estimates of the empirical IV for different \(n\) and \(h\), and a formatted output of the estimated parameters \(C\), \(\beta\), and \(delta\) so that the details of this experiment can be written to a file. If the user wishes to display output at runtime as well, simply set displayExec to true<>.

The way this method works is by calling parametersIVPreprocess, parametersIVFormatHead, parametersIVComputeRegData, parametersIVComputeCoefficients, and parametersIVFormatCoefficients sequentially.

rqmcPtsthe RQMC point sets to construct the observation.
modelthe underlying model, from which the observations are obtained.
mthe number of independent repetitions.
dethe histogram.
hArraythe values for \(h\) of the testing region.
evalPointsthe evaluation points used to estimate the empirical IV.
Returns
a formatted String containing all the information about this parameter estimation.
